交换机远程访问配置

发表于 2019-03-27 更新于 2019-09-07 分类于 计算机网络

前言

刚学，简略记录一下交换机的IP、网关、访问密码、远程控制的配置过程。

由于没有实物，这里拿 Cisco Packet Tracer 的模拟环境配图。

模式切换命令

模式 命令 提示符
用户模式 Switch>
特权模式 enable Switch#
全局配置模式 config terminal Switch(config)#
端口配置模式 interface fastethernet 端口号 Switch(config-if)#
控制设置模式 line control 控制口 Switch(config-line)#
线路设置模式 line vty 线路口 Switch(config-line)#
VLAN 设置模式 Vlan database Switch(vlan)#

连接交换机与 PC

如图，PC0 的 RS232 接口连接交换机的 Console 接口，PC1 的 FastEthernet 接口连接交换机的 FastEthernet 接口。

PC0 上的配置

因为 PC0 通过串口协议与交换机进行通讯的，所以打开终端选择相应的串口就能看见消息了。

临时约定：以##开头的单独成行的语句为注释。

注：实际操作中要打开终端选择相应的串口。

Windows XP 自带超级终端软件：开始->附件->通讯->超级终端。

其他系统如果没有超级终端也可以用其他支持串口通讯的软件，比如Putty。

PC1 上的配置

IP 地址配置

设置 PC1 的 IP 地址为192.168.1.2，子网掩码255.255.255.0，默认网关192.168.1.1。其中，网关地址就是交换机的地址。

注：实际操作的时候在网络适配器的设置里更改 IPv4 协议的配置。

Ping 测试连通性

注：在实际操作的时候是在 CMD 中输入命令。可以在开始->运行->输入CMD->打开CMD窗口。

打开命令提示符，输入命令ping 192.168.1.1，检查 PC1 与交换机之前的连通性，其中192.168.1.1是交换机的 IP 地址。

如果能连通就下一步，不能连通就先排错。

Telnet 连接

注：在实际操作的时候是在 CMD 中输入命令。可以在开始->运行->输入CMD->打开CMD窗口。

打开命令提示符，输入命令telnet 192.168.1.1。

连接成功后输入三次密码，分别是之前在 PC0 上设置的三个密码，输入正确后即可看到Switch>的回显。

交换机重置

后记

在配置交换机的密码之前只能通过串口（Console）来配置交换机。

PC0 通过串口协议与交换机通讯，打开终端能看到信息。

PC1 通过 Telnet 协议与交换机进行通讯，而 Telnet 协议基于 TCP/IP，所以要先配置好交换机和 PC1 的 IP 地址，然后通过命令提示符（CMD）运行 telnet 命令来与交换机进行通讯。

IP 地址可灵活配置，只要保证交换机和 PC 在同一网段，能相互访问即可。
